# 드래그 앤 드롭 업로드 설계

## 목표

기존 업로드 영역에 스프라이트 시트 PNG를 드래그 앤 드롭으로 바로 올릴 수 있게 하되, 현재의 클릭 업로드 흐름은 그대로 유지한다.

## 제품 요구사항

현재 UI는 파일 선택창을 통한 업로드만 지원한다. 기능적으로는 충분하지만, 스프라이트 시트를 반복해서 시험하는 도구 특성상 업로드 마찰이 크다. 따라서 기존 업로드 영역이 PNG 드롭 타겟으로도 동작해야 한다.

## 권장 접근

별도 업로드 박스를 추가하지 않고, 현재의 업로드 dropzone을 그대로 확장한다.

이 방식은 지금 UI를 거의 바꾸지 않으면서도 상호작용만 넓혀준다. 같은 영역에서 아래 두 동작을 모두 지원하면 된다.

- 클릭해서 PNG 선택
- PNG를 드래그해 영역 위에 놓아 바로 처리

## 사용자 경험

예상 흐름은 다음과 같다.

1. 사용자가 PNG를 기존 업로드 카드 위로 드래그한다.
2. 카드가 drag-over 강조 상태로 바뀐다.
3. 사용자가 파일을 놓는다.
4. 기존 파일 선택 업로드와 동일한 처리 흐름이 실행된다.

기존 클릭 업로드 동작은 절대 깨지면 안 된다.

## 검증 규칙

- PNG 파일만 허용
- 지원하지 않는 파일 형식은 무시
- 여러 파일이 드롭되면 첫 번째 PNG만 사용
- 유효한 PNG가 없으면 조용히 실패하지 말고, 현재 상태/에러 표시 방식으로 사용자에게 알려준다

## UI 동작

기존 업로드 컴포넌트가 drag 상태를 시각적으로 보여줘야 한다.

최소 요구 상태:

- 기본 상태
- drag-over 강조 상태
- 현재처럼 처리 중/비활성 상태 유지

강조 상태는 분명하게 보여야 하지만, 새로운 업로드 모델이 하나 더 생긴 것처럼 느껴지면 안 된다.

## 구현 범위

주요 파일:

- `src/app.tsx`
- `src/styles.css`

예상 변경:

- 기존 업로드 영역에 drag 이벤트 핸들러 추가
- 기존 `processFile(...)` 흐름 재사용
- 작은 `isDragOver` 상태 추가
- drag-over 강조용 CSS 클래스 추가

## 하위 호환성

이 기능은 아래를 깨뜨리면 안 된다.

- 클릭 업로드 동작
- 현재 auto/manual 처리 흐름
- 현재 상태 메시지와 에러 처리 방식
- 현재 브라우저 수동 테스트 및 Playwright 확인 흐름

## 품질 기준

사용자가 클릭 업로드와 드래그 앤 드롭 업로드를 같은 영역에서 모두 쓸 수 있고, drag-over 시 명확한 시각 피드백을 받으며, 처리 결과가 기존과 동일하면 성공이다.
